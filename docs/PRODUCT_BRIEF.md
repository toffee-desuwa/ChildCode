# ChildCode — 产品定义

## 产品定位

一个面向 8–12 岁孩子的 AI 时代表达启蒙工具。孩子不用打字写 prompt，而是通过拖拽积木组合对象、动作、场景、风格，生成图片，并通过只改一个块再比较结果的闭环，理解输入结构会影响 AI 输出。

我们在探索一种新的儿童 AI 表达启蒙方式。

## 用户角色

### 孩子（8–12 岁，主要用户）
- 可独立探索，不需要家长全程陪同
- 通过拖拽白名单积木组合表达，无需打字
- 通过"改一个块 → 再生成 → 看差异"建立因果理解

### 家长（配置与安全角色）
- 负责首次配置（API key 等）
- 控制使用额度
- 最终安全兜底

## MVP 核心闭环

闭环不是"出一张图就结束"，而是：

1. 孩子拖 3–5 个积木块，生成第一张图
2. 替换其中 1 个块，生成第二张图
3. 直观看到两张图的差异
4. 理解：不同的输入结构 → 不同的 AI 输出

## 四类基础表达（第一版）

| 类别 | 说明 | 示例 |
|------|------|------|
| 对象 | 图中的主体 | 猫、机器人、小女孩 |
| 动作 | 主体在做什么 | 奔跑、飞翔、画画 |
| 场景 | 发生在哪里 | 森林、太空、海底 |
| 风格 | 画面呈现方式 | 水彩、像素风、油画 |

## 学习目标

孩子在使用中自然习得：
- **结构化表达**：把想法拆成对象 + 动作 + 场景 + 风格
- **因果推理**：改变一个输入，观察输出如何变化
- **AI 素养启蒙**：AI 的输出取决于你怎么描述，而不是魔法

## 安全边界

- **完全白名单**：积木内容由系统预设，不开放自由文本输入
- **白名单显著降低风险**：积木内容经预先审核，大幅减少不当内容可能性，但不宣称零风险；最终由家长监护与平台机制共同兜底
- **家长额度控制**：限制 API 调用次数 / 费用
- **无社交功能**：不涉及用户间交互

## 技术表示层原则

采用双层表示架构：

```
Blockly 积木 → 结构化 JSON（真相层）→ prompt（派生执行层）→ 图片 API
```

- **结构化 JSON 是真相层**：所有逻辑基于 JSON，不基于 prompt 文本
- **prompt 是派生层**：由系统根据 JSON 自动生成，孩子不接触 prompt
- Blockly 不直接拼最终 prompt

## 非目标（MVP 明确不做）

- 自由文本输入
- 课程系统 / 教学引导流程
- Harness / 测试管理 UI
- 商业化 / 付费 / 账户体系
- 多语言支持
- 社交 / 分享功能
- 直接教授 prompt 术语、复杂 prompt engineering 技巧或自由文本写法
