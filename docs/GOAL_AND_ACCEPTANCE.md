# ChildCode — Goal / Non-goals / Acceptance

## Goal

1. 让 8–12 岁孩子通过拖拽白名单积木（对象、动作、场景、风格），无需打字，即可组合结构化输入并生成图片
2. 实现"改一个块 → 再生成 → 比较差异"的核心闭环，让孩子直观理解输入结构如何影响 AI 输出
3. 家长可完成首次配置（API key）、额度控制与最终安全兜底

## Non-goals

1. 不开放自由文本输入，孩子只能在白名单积木中选择
2. 不做课程系统、教学引导流程或闯关机制
3. 不做 harness / 测试管理 UI
4. 不做商业化、付费、账户体系
5. 不做多语言（MVP 只做中文）
6. 不做社交、分享、用户间交互功能
7. 不直接教授 prompt 术语、复杂 prompt engineering 技巧或自由文本写法
8. 不做 prompt 直接拼接——必须走结构化 JSON 中间层

## Acceptance（验收标准）

以下每条均为可检查的通过条件：

### A1 — 积木拖拽与结构化输出
- [ ] 页面展示四类白名单积木：对象、动作、场景、风格
- [ ] 孩子可拖拽 3–5 个积木块到工作区
- [ ] 拖拽完成后，系统生成结构化 JSON（非 prompt 文本）
- [ ] JSON 至少包含 `subject`、`action`、`scene`、`style` 四类核心信息
- [ ] MVP 默认只使用这四类核心信息参与 prompt 派生

### A2 — 图片生成
- [ ] 系统将结构化 JSON 转换为 prompt（派生层），孩子不可见 prompt
- [ ] 调用图片生成 API，返回一张图片并展示给孩子

### A3 — 对比闭环
- [ ] 孩子可在已有积木组合中替换 1 个块
- [ ] 替换后可再次生成图片
- [ ] 两张图片（替换前 / 替换后）同屏展示，孩子可直观对比差异

### A4 — 家长配置与安全
- [ ] 家长可完成 API key 与使用额度的最小配置
- [ ] 额度耗尽时，系统阻止继续生成并给出提示

### A5 — 安全边界
- [ ] 孩子无法输入自由文本，所有输入来自白名单积木
- [ ] 白名单积木内容经预先审核
- [ ] 不存在绕过白名单的操作路径
